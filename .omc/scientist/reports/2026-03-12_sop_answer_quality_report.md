## [FINDING 1] 인용(Citation) 가시성이 매우 낮다

200자 미리보기 안에 인용 표기를 포함하는 답변은 전체의 30.4%에 불과하다.
나머지 69.6%(55건)는 200자 안에서 인용 표기가 보이지 않는다.

[STAT:n] n=79
[STAT:effect_size] 69.6% (55/79)가 preview 내 무인용; 기대치(100%)와의 괴리 크다

**인용 형식 불일치(3건):** 동일 답변 내에서 `[1]` 스타일과 `【[1]】` 스타일이 혼용된 사례가 3건 확인된다(idx=0, 64, 69).
프롬프트에는 `[1]` 형식을 명시하고 있으나, 모델이 자의적으로 `【】` 형식을 생성하는 경우가 있다.

추가 관찰된 형식:
- `[N]` 표준 형식: 13건
- `【N】` / `【N†pX】` 특수 형식: 14건 (프롬프트 미지정 형식)
- `【[N]】` 이중 래핑 혼합: 3건

## [FINDING 2] 답변 포맷이 매우 비일관적이다

동일한 v1 프롬프트를 사용하더라도 출력 포맷이 케이스마다 크게 다르다.

[STAT:n] n=79
[STAT:effect_size] 51.9%(41건)가 2가지 이상 포맷 혼용 (테이블 + 이모지 번호 등)

| 포맷 유형 | 건수 | 비율 |
|-----------|------|------|
| 이모지 번호 (1️⃣2️⃣) | 48 | 60.8% |
| 테이블 (\|col\|) | 46 | 58.2% |
| 불릿 포인트 (-) | 19 | 24.1% |
| 일반 텍스트 | 6 | 7.6% |
| 숫자 목록 (1.) | 4 | 5.1% |

v1 프롬프트는 "단계별 절차(번호 목록)"를 요구하지만, 모델은 테이블, 이모지, 불릿, 평문을 혼용해서 자유롭게 선택한다.
v2 프롬프트(EN/JA/ZH)는 Purpose→Pre-check→Procedure→Post-check→Cautions→References의 구조를 강제하지만, **한국어 v2(`setup_ans_v2.yaml`)는 v1과 동일**하여 구조 강제가 없다.

## [FINDING 7] 프롬프트 v1 vs v2 구조적 차이와 영향

`setup_ans_v1.yaml` (KO): 자유형 지시 → 출력 포맷 비일관적
`setup_ans_v2.yaml` (KO): v1과 **완전히 동일** → 구조 개선 없음

반면 `setup_ans_en_v2.yaml`, `setup_ans_ja_v2.yaml`, `setup_ans_zh_v2.yaml`는 모두:
- 목적(Purpose), 사전 확인(Pre-check), 절차(Procedure), 사후 확인(Post-check), 주의사항(Cautions), 참고문헌(References) 6개 섹션 강제
- 빈 섹션 생략 허용
- `[N]` 형식 인용을 절차 각 단계에 필수화

**한국어 v2가 v1과 동일한 것은 명백한 누락이다.** 현재 79건 평가는 어떤 프롬프트 버전으로 생성되었는지 불명확하나, 포맷 비일관성은 v1 수준의 지시를 받고 있음을 시사한다.

## [LIMITATION]

1. **200자 truncation 한계:** answer_preview는 200자로 절단되어 있어 인용 위치, 전체 절차 완성도, 참고문헌 목록의 존재 여부를 판단할 수 없다. 실제 전체 답변 품질은 이 분석보다 높을 수 있다.
2. **Heuristic 기반 품질 측정:** "안전 키워드 포함"이 실제 안전 내용의 적절성을 보장하지 않는다. 사람의 판단이 필요한 부분이다.
3. **언어 분류 단순화:** 한국어/영어 비율로 언어를 분류했으나, 도메인 기술 용어의 영어 혼용은 정상 패턴이므로 "혼합"이 반드시 문제가 아니다.
4. **프롬프트 버전 불확실성:** 79건 답변이 v1과 v2 중 어느 프롬프트로 생성됐는지 데이터에서 확인 불가.
5. **n=79 소규모:** 특정 idx의 개별 결함이 통계적으로 유의미한지 단정하기 어렵다.
