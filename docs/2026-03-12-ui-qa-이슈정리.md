# UI QA 이슈 정리

## 목적

화면 QA 중 확인된 UI 이슈 2건을 정리한다.  
이 문서는 구현자가 바로 수정 방향을 이해하고 작업할 수 있도록, 문제 상황, 기대 동작, 추정 원인, 수용 기준을 함께 정리한 문서다.

---

## 이슈 1. 답변 종료 후 불필요한 장비 검색 안내 노출

### 현재 문제

답변이 끝난 뒤 아래 안내가 다시 노출된다.

```text
장비를 자동으로 파싱하지 못했습니다. 기기를 검색하시겠습니까?
```

사용자 입장에서는 이미 답변이 완료된 뒤이므로 이 안내가 어색하다.  
특히 SOP/절차검색이나 일반 질의에서 답변이 정상적으로 끝났는데도 다시 장비 검색 유도가 뜨면 흐름이 끊긴다.

### 기대 동작

- 답변이 정상적으로 완료된 뒤에는 위 안내가 뜨지 않아야 한다.
- 장비 검색 유도는 질문 처리 전에 필요한 경우에만 보여야 한다.
- 이미 답변이 생성된 뒤에는 `device search prompt`를 후행으로 띄우지 않아야 한다.

### 추정 원인

가능성은 아래 중 하나다.

1. `auto_parse` 실패 상태가 answer 이후까지 남아 있고, 후속 렌더링에서 다시 panel이 뜸
2. `pendingGuidedSelection` 또는 유사한 interrupt state가 answer 완료 후 정리되지 않음
3. answer 완료 후에도 `suggest_additional_device_search` 같은 플래그를 그대로 UI에서 렌더링함

### 수정 방향

- answer가 완료되면 장비 선택/검색 관련 pending state를 정리
- answer 이후에는 device-search 관련 panel/message를 다시 띄우지 않도록 guard 추가
- "질문 입력 전 단계에서만 허용" / "answer 이후 금지" 규칙을 UI 렌더링 레벨에서 명확히 분리

### 수용 기준

1. 답변 완료 후 `장비를 자동으로 파싱하지 못했습니다...` 문구가 더 이상 표시되지 않는다.
2. 장비 검색 panel은 답변 이전의 입력 유도 단계에서만 표시된다.
3. 대화 이력 재로딩 시에도 과거 answer 메시지 아래에 장비 검색 안내가 다시 나타나지 않는다.

## 이슈 2. 이미지 문서 클릭 시 세로 모니터에서는 열리지만 가로 모니터에서는 페이지가 열리지 않음

### 현재 문제

검색된 문서가 이미지로 표시될 때, 이미지를 클릭하면:

- 세로 모니터 환경에서는 페이지가 정상 표시됨
- 가로 모니터 환경에서는 페이지가 뜨지 않거나 preview가 정상 렌더링되지 않음

즉, 화면 비율/레이아웃 조건에 따라 문서 preview 동작이 달라진다.

### 기대 동작

- 세로/가로 모니터 여부와 관계없이, 이미지 문서를 클릭하면 항상 preview modal 또는 페이지 viewer가 정상적으로 열려야 한다.
- viewport 폭이 달라도 동일한 문서와 동일한 페이지가 안정적으로 표시되어야 한다.
- 반응형 레이아웃 때문에 클릭 이벤트나 preview 상태가 깨지면 안 된다.

### 추정 원인

가능성은 아래와 같다.

1. 가로 레이아웃에서 preview modal/container의 width/height 계산이 잘못되어 실제로는 열렸지만 화면 밖으로 밀림
2. 클릭 시 전달되는 `docIndex/pageIndex`와 preview image array 매핑이 화면 폭에 따라 달라짐
3. desktop wide layout에서 side panel / modal z-index 또는 overflow 이슈가 발생
4. 가로 모니터에서 fit-to-page / object-fit / max-height 계산이 잘못되어 빈 화면처럼 보임

### 수정 방향

- 문서 클릭 시 생성되는 preview image list와 selected index 매핑을 화면 크기와 무관하게 고정
- modal/container의 `max-width`, `max-height`, `overflow`, `z-index`를 반응형 기준으로 재검토
- 가로 모니터에서도 기본적으로 "페이지 전체가 보이는" fit-to-page 동작을 우선 적용
- 세로/가로 viewport 각각에서 동일한 클릭 시나리오를 직접 검증

### 수용 기준

1. 가로 모니터 환경에서도 이미지 문서를 클릭하면 preview가 정상적으로 열린다.
2. 클릭한 문서/페이지와 실제로 열린 preview 페이지가 일치한다.
3. modal이 화면 밖으로 밀리거나 빈 영역만 보이지 않는다.
4. 세로/가로 해상도 모두에서 동일한 동작을 보인다.

## 구현 시 확인 포인트

- answer 완료 시점의 UI state 정리 여부
- device search panel 렌더링 조건
- image preview modal open 조건
- `docIndex`, `pageIndex`, `previewImages` 매핑
- wide viewport에서 modal sizing / overflow / z-index

---

## 결론

이번 QA 이슈의 핵심은 다음 두 가지다.

1. **답변 이후에는 장비 검색 안내를 다시 띄우지 않기**
2. **가로 모니터에서도 이미지 문서 preview가 항상 열리도록 보장하기**

두 이슈 모두 기능 자체보다 UI 상태 관리와 반응형 렌더링 조건 정리가 핵심이다.
