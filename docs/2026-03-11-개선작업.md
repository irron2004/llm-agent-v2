## 3. 이슈 플로우 수정 (Task 5 재설계)

### 현재 구현의 문제

현재 `task_mode=issue`로 검색하면:
- 답변 없이 바로 "이슈 사례를 선택하세요" interrupt가 표시됨
- 사용자는 어떤 맥락인지 모른 채 선택해야 함
- UX가 나쁨

### 수정할 플로우

```
검색 → answer_node (이슈별 요약 답변, 번호 리스트 형식)
    → 답변 표시
    → "더 확인하고 싶은 이슈가 있나요?" (yes/no)
    → no → 종료
    → yes → 검색된 문서 목록 표시 → 사용자 선택
         → 선택 문서 기반 상세 답변 생성
         → "관련 SOP를 확인할까요?" (yes/no)
         → no → "다른 이슈도 볼까요?" (yes/no) → yes면 문서 목록으로
         → yes → SOP 검색 → SOP 답변 표시
              → "다른 이슈도 볼까요?" (yes/no) → yes면 문서 목록으로
```

### 설계 포인트

1. **먼저 요약 답변을 보여줌** — 이슈별 번호 리스트로 구조화
   ```
   검색된 이슈 사례:
   1. [40129396] LP1 Mapping Arm Open Alarm — 원인: 센서 이상, 조치: 센서 교체
   2. [40159486] RFID Read 불가 — 원인: CTC GUI 설정, 조치: 재설정
   더 확인하고 싶은 이슈가 있나요? (yes/no)
   ```

2. **1건이어도 항상 yes/no 확인** — 검색 결과가 마음에 안 들 수 있으므로 사용자에게 선택권 부여

3. **SOP는 별도 질문** — 토큰 부족 방지 + 원하지 않는 이슈일 수 있으므로 별도 확인

4. **"다른 이슈도 볼까요?" 루프** — 상세 답변 후 다시 문서 목록으로 돌아갈 수 있음

### 변경 대상 파일

- `backend/llm_infrastructure/llm/langgraph_agent.py` — issue_step1 노드 수정 (먼저 answer 생성 후 yes/no interrupt)
