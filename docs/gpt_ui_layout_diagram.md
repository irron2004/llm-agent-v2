# GPT UI 레이아웃 다이어그램 및 시각적 가이드

## 1. 레이아웃 구조 비교

### 현재 레이아웃 (Before)

```
┌─────────────────────────────────────────────────────────┐
│  Header (고정)                                         │
│  [PE Agent] [Chat] [Search] [Parsing] [Retrieval Test] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              Main Content Area                          │
│              (전체 너비 사용)                           │
│                                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### GPT 스타일 레이아웃 (After)

```
┌──────────┬──────────────────────────────────┬──────────┐
│          │                                  │          │
│  Left    │      Main Content Area           │  Right   │
│ Sidebar  │      (최대 너비 제한)            │ Sidebar  │
│          │                                  │ (선택적) │
│ 288px    │                                  │  320px   │
│          │                                  │          │
│ - 로고   │  - 페이지 헤더                   │ - 작업   │
│ - 새채팅 │  - 콘텐츠                        │   정보   │
│ - 검색   │  - 입력 영역                     │ - AI     │
│ - 메뉴   │                                  │   생각   │
│ - 히스토리│                                  │          │
└──────────┴──────────────────────────────────┴──────────┘
```

## 7. 인터랙션 플로우

### 새 채팅 시작 플로우

```
사용자 클릭: "새 채팅"
    ↓
[왼쪽 사이드바] 새 채팅 버튼
    ↓
채팅 세션 초기화
    ↓
중앙 콘텐츠 영역으로 포커스 이동
    ↓
입력 영역 활성화
```

### 페이지 전환 플로우

```
사용자 클릭: 네비게이션 메뉴
    ↓
[왼쪽 사이드바] 메뉴 항목 클릭
    ↓
라우터 네비게이션
    ↓
해당 페이지 컴포넌트 렌더링
    ↓
활성 메뉴 항목 하이라이트
```

### 채팅 히스토리 플로우

```
사용자 클릭: 히스토리 항목
    ↓
[왼쪽 사이드바] 히스토리 항목 클릭
    ↓
해당 채팅 세션 로드
    ↓
중앙 콘텐츠에 메시지 표시
    ↓
해당 히스토리 항목 하이라이트
```

## 8. 컴포넌트 계층 구조

```
Layout (최상위)
├── LeftSidebar
│   ├── SidebarHeader (로고 + 새 채팅)
│   ├── SidebarSearch (검색 입력)
│   ├── SidebarNav (네비게이션 메뉴)
│   └── SidebarHistory (채팅 히스토리)
│
├── MainContent
│   └── Outlet (라우터 콘텐츠)
│       ├── ChatPage
│       ├── SearchPage
│       ├── ParsingPage
│       └── RetrievalTestPage
│
└── RightSidebar (선택적)
    ├── RightSidebarHeader
    └── RightSidebarContent
```

## 9. 스페이싱 가이드

### 패딩 및 마진

```
사이드바:
- 외부 패딩: 16px (--spacing-lg)
- 내부 아이템 간격: 4px (--spacing-xs)
- 섹션 간격: 12px (--spacing-md)

중앙 콘텐츠:
- 외부 패딩: 24px (--spacing-xl)
- 메시지 간격: 24px (--spacing-xl)
- 입력 영역 상단 여백: 16px (--spacing-lg)
```

## 12. 접근성 고려사항

### 키보드 네비게이션

- Tab: 사이드바 메뉴 간 이동
- Enter: 메뉴 항목 선택
- Escape: 사이드바 닫기 (모바일)
- Arrow Up/Down: 히스토리 목록 탐색

### ARIA 레이블

```html
<nav aria-label="주요 네비게이션">
  <button aria-label="새 채팅 시작">새 채팅</button>
</nav>

<aside aria-label="채팅 히스토리">
  <h2>최근 대화</h2>
</aside>
```

## 13. 성능 최적화 전략

### 가상 스크롤
- 채팅 히스토리가 많을 경우 가상 스크롤 적용
- react-window 또는 react-virtualized 사용

### 메모이제이션
- 사이드바 메뉴 아이템 메모이제이션
- 히스토리 목록 메모이제이션

### 코드 스플리팅
- 각 페이지를 별도 청크로 분리
- 사이드바 컴포넌트 지연 로딩
