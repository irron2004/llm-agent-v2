# Device Suggestion Follow-up Flow Spec (2026-01-29)

## 목적
장비명이 명시되지 않은 질문에 대해 먼저 답변을 제공하고, 검색된 문서의 device_name 분포를 기반으로 장비를 제안한다. 사용자가 숫자로 선택하면 동일 질문을 해당 장비 필터로 재검색한다.

## 범위
1. 장비 지정 여부 판단 기준은 auto_parse 결과만 사용한다.
2. 추천 장비는 검색된 문서 50개 기준, device_name count 내림차순 전체 나열이다.
3. 선택 입력은 텍스트 숫자만 우선 지원한다.

## 사용자 흐름
1. 사용자가 장비명을 명시하지 않은 질문을 입력한다.
2. 시스템은 일반 RAG 흐름으로 답변을 생성한다.
3. 답변 하단에 추천 장비 리스트를 번호로 표시한다.
4. 사용자가 숫자만 입력하면 해당 장비를 선택한다.
5. 직전 질문을 동일하게 유지하고 device filter만 적용하여 재검색한다.

## 노출 규칙
1. auto_parse.device 또는 auto_parse.devices가 존재하면 추천 리스트를 표시하지 않는다.
2. retrieved_docs가 비어 있으면 추천 리스트를 표시하지 않는다.
3. 추천 리스트는 count 내림차순 전체 노출한다.

## 데이터 정의
### SuggestedDevice
- name: string
- count: number

### AgentResponse 확장
- suggested_devices: SuggestedDevice[] | null

## 백엔드 변경
### 집계 로직
1. display_docs 또는 docs에서 metadata.device_name을 수집한다.
2. 비어있거나 의미 없는 값은 제외한다.
3. count 내림차순으로 정렬한다.
4. 전체 목록을 suggested_devices로 응답에 포함한다.

### 필터링 정책
1. device_name이 빈 문자열이거나 None인 경우 제외한다.
2. "ALL", "etc" 같은 더미 값은 제외한다.
3. trim 이후 빈 문자열이면 제외한다.

### 응답 스키마
AgentResponse에 suggested_devices를 추가한다.

## 프론트엔드 변경
### 렌더링
1. assistant 답변 아래에 추천 장비 목록을 렌더링한다.
2. 리스트는 1부터 번호를 붙여 문자열로 표시한다.

### 숫자 선택 처리
1. 사용자가 입력한 메시지가 숫자만 포함하면 선택 처리한다.
2. 마지막 assistant 메시지의 suggested_devices에서 해당 인덱스를 찾는다.
3. 선택이 유효하면 실제 요청 메시지는 직전 질문으로 대체한다.
4. 요청 시 filter_devices에 선택된 장비를 넣고 auto_parse=false로 보낸다.

### 표시 메시지
1. UI에는 사용자가 입력한 숫자 그대로 표시한다.
2. 서버 요청 메시지는 내부적으로 직전 질문을 사용한다.

## 상태 전달
1. 마지막 질문 텍스트를 프론트 상태로 유지한다.
2. 마지막 suggested_devices를 프론트 상태로 유지한다.

## 예외 처리
1. 숫자가 범위를 벗어나면 일반 질문으로 처리한다.
2. suggested_devices가 없으면 숫자 입력도 일반 질문으로 처리한다.
3. 직전 질문이 없으면 숫자 입력을 무시하고 안내 메시지를 반환한다.

## 로깅
1. suggested_devices 생성 여부와 개수를 로그로 남긴다.
2. 숫자 선택 입력 감지 여부를 로그로 남긴다.
3. 선택된 장비명과 재검색 여부를 로그로 남긴다.

## 테스트 시나리오
1. 장비 미지정 질문 후 추천 리스트 표시.
2. 장비 지정 질문은 추천 리스트 미표시.
3. 숫자 선택 입력으로 재검색 수행.
4. 숫자 범위 초과 시 일반 질문 처리.
5. retrieved_docs 없는 경우 추천 미표시.

## 롤아웃
1. 백엔드 응답 확장 후 프론트 렌더링 연결.
2. 숫자 선택 처리 기능을 마지막 단계에서 활성화.
